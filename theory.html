Theory of Computation — theory section

Connects the form validators to formal languages and finite automata. A plain-text copy lives in docs/theory.txt.

1. Definition of regular expressions

A regular expression (regex) is a finite description of a set of strings over an alphabet. In formal language theory, the class of languages denoted by regular expressions is exactly the regular languages (also called Type-3 in the Chomsky hierarchy). Practical regex engines add conveniences (e.g. {2,30}) that still correspond to regular languages when used as bounded repetition over a finite alphabet, as in this project.

2. Formal language for each field

Full name

L_name = { w in {A-Z, a-z, space}* | 2 <= |w| <= 30 and w matches ^[A-Za-z ]{2,30}$ }

Date of birth (syntactic)

L_dob = { w | w = DD/MM/YYYY with '/' separators, DD in 01-31, MM in 01-12, YYYY any four digits }

Exam code

L_exam = { w | w is a valid exam code: exactly 2 uppercase Latin letters followed by 3 decimal digits }

Hall ticket

L_hall = { w | w = alpha ++ beta, |alpha|=3, alpha in {A-Z}^3, |beta|=5, beta in {0-9}^5 }

Subject code

L_subj = { w | w = alpha ++ beta, alpha in {A-Z}^2 U {A-Z}^3, |beta|=3, beta in {0-9}^3 }

Uploaded filename

L_file = { f | f ends with ".pdf", ".jpg", or ".png" (case-insensitive in implementation) }

3. Why each validation is a regular language

Regular languages are closed under union, concatenation, and Kleene star, and are equivalently those accepted by some finite automaton. Each field uses only:

  • finite alphabets (letters, digits, slash, dot),
  • concatenation of fixed-length blocks,
  • bounded repetition (name length 2–30),
  • finite alternation (|) inside day/month alternatives.

Therefore each L_* is regular, and each JavaScript RegExp used here describes a regular language (no unbounded counting beyond what a DFA can track in finitely many states).

4. DFA / NFA sketches

4a. Mobile-like digit block — exactly five digits

Phone-style identifiers are often digit strings of fixed length. Here: a DFA for exactly five digits (the numeric suffix of a hall ticket). States q0..q5; q5 is the only accept state; each transition reads one digit 0–9. 

States: q0 (start), q1, q2, q3, q4, q5 (accept)

For each i in 0..4: qi --(digit)--> q(i+1)

Reject if string ends before q5 or continues after q5 with another digit.

ASCII:
   d       d       d       d       d
(q0)-->(q1)-->(q2)-->(q3)-->(q4)-->((q5))

4b. Exam code — two letters then three digits

Template LLDDD yields a small deterministic chain of states.

S0 --[A-Z]--> S1 --[A-Z]--> S2 --[0-9]--> S3 --[0-9]--> S4 --[0-9]--> S5 (ACCEPT)

Reject on wrong symbol class or wrong length.

Example: CS101
  S0 -C-> S1 -S-> S2 -1-> S3 -0-> S4 -1-> S5 accept

4c. NFA remark

Any DFA is trivially an NFA. For languages built with |, an NFA can branch; subset construction produces an equivalent DFA. Our patterns are already close to deterministic chains plus finite unions (e.g. day/month digit ranges), so a single DFA exists without exponential blow-up for these toy alphabets.
